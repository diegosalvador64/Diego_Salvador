HTML formularios

El elemento <form>

El Elemento HTML<form>define una forma que se utiliza para recolectar información del usuario:

Un formulario HTML contiene elementos de formulario .

Los elementos del formulario son diferentes tipos de elementos de entrada, como campos de texto, casillas de verificación, botones de radio, botones, y más.

El atributo action

El action atributo define la acción que debe realizarse cuando se envía el formulario.

Normalmente, los datos del formulario se envía a una página web en el servidor cuando el usuario hace clic en el botón de envío.

En el ejemplo anterior, los datos del formulario se envía a una página en el servidor llamado "/action_page.php". Esta página contiene un script del lado del servidor que gestiona los datos del formulario:

<form action="/action_page.php">

<form action="mailto:correo@gmail.com">

<form action="" onsubmit="return validarFormulario()">

El atributo de destino

El targetatributo especifica si el resultado presentado se abrirá en una nueva pestaña del navegador, un marco, o en la ventana actual.

El valor por defecto es " _self" que significa que el formulario se haya presentado en la ventana actual.

Para que el resultado formulario se abre en una nueva pestaña del navegador, utilice el valor " _blank":

<form action="/action_page.php" target="_blank">

Otros valores son legales " _parent", " _top", o un nombre que representa el nombre de un iframe.

El atributo del método

El method atributo especifica el método HTTP ( GET o la POST ) para ser utilizado cuando la presentación de los datos del formulario:

<form action="/action_page.php" method="get">

<form action="/action_page.php" method="post">

El método por defecto al presentar los datos del formulario es GET.

Sin embargo, cuando se utiliza GET, los datos de formularios presentados serán visibles en el campo de dirección de la página :

/action_page.php?firstname=Mickey&lastname=Mouse

Notas sobre GET:

  • Anexa forma-datos en el URL en el nombre de pares / valor
  • La longitud de un URL es limitado (aproximadamente 3000 caracteres)
  • Nunca utilice GET para enviar datos sensibles! (Serán visibles en la URL)
  • Útil para el envío de formularios donde un usuario desea marcar el resultado
  • GET es mejor para los datos no seguros, como las cadenas de consulta en Google

¿Cuándo utilizar la POST?

Siempre use POST si los datos del formulario contiene información confidencial o personal. El método POST no muestra los datos del formulario presentado en el campo de dirección de la página.

Aquí está la lista de todos los <form>atributos:

Atributo Descripción
accept-charset Especifica el conjunto de caracteres utilizado en el formulario enviado (por defecto: la página de juego de caracteres).
action Especifica una dirección (URL) donde enviar el formulario (por defecto: la página de remitentes).
autocomplete Especifica si el navegador debe autocompletar la forma (por defecto: activado).
enctype Especifica la codificación de los datos presentados (por defecto: es codificada en URL).
method Especifica el método HTTP utilizado al presentar el formulario (por defecto: GET).
name Especifica un nombre usado para identificar la forma (para el uso de DOM: document.forms.name).
novalidate Especifica que el navegador no se debe validar el formulario.
target Especifica el destino de la dirección en el atributo de acción (por defecto: _self).
